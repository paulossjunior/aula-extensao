    Aula 02 — Prototipação Rápida com IA e Validação de Requisitos¶

    Data: 16/03/2026 | Horário: 11h00 | Local: Sala 207

    Introdução¶

    Nesta aula, avançamos da identificação do problema para a materialização da solução. O objetivo é utilizar ferramentas de Inteligência Artificial Generativa para criar protótipos funcionais em tempo recorde, permitindo a validação imediata dos requisitos levantados no PRD da aula anterior.

    Discovery do Projeto¶

    A prototipação com IA mudou drasticamente a velocidade do ciclo Build-Measure-Learn. Em vez de dias desenhando telas, podemos gerar interfaces funcionais em minutos para testar a proposta de valor.

    Ferramentas de Prototipação com IA¶

    • Figma Maker (Online / Free): A ferramenta principal desta aula permite a geração rápida de wireframes e telas baseadas em prompts diretamente no navegador. Isso agiliza a tradução de requisitos para a primeira interface sem a necessidade de baixar softwares complexos.
    • Alternativas para código gerado: Se o foco técnico for código, o v0.dev e o Bolt.new podem mapear requisitos para interfaces Tailwind ou focar no Full-Stack. Nesta aula, o foco é o Figma Maker Online para validação de produto via telas rápidas.

    Preparação Importante

    Certifique-se de ter o seu PRD (Aula 01) em mãos. O aluno deve trazer os requisitos macros do produto devidamente listados. Eles serão o "input" principal para a geração das telas no Figma Maker hoje.

    Tarefas¶

    Tarefa 1 — Do PRD aos Requisitos Macros (Estruturando o Prompt)¶

    Duração estimada: 20 min Formato: Individual ou Dupla

    Sua missão é traduzir o problema e o PRD para um direcional que a IA consiga entender. Liste as 3 funcionalidades core (principais) e traduza cada uma em seus requisitos macros.

    Escreva um Prompt Estruturado para usar no Figma Maker: - Contexto: Quem é o usuário e qual a proposta de valor do produto. - Tarefa (Telas-alvo): Que telas específicas a IA deve construir (ex: uma tela inicial, um dashboard ou um fluxo de onboarding). - Requisitos Macros do Produto: A listagem clara das seções ou conteúdos que as telas precisam exibir para atender ao PRD.

    Entregável: Uma listagem com os requisitos macros do produto e o texto do prompt estruturado.

    Tarefa 2 — Geração do Protótipo com Figma Maker¶

    Duração estimada: 25 min Formato: Prático no computador

    Nesta etapa, você utilizará o plugin / ferramenta de IA do Figma para criar sua primeira versão tangível.

    1. Acesse o Figma Maker (Online) no navegador e insira o seu prompt contendo os requisitos macros da Tarefa 1.
    2. Gere os primeiros frames. Interaja modificando cores e ajustando componentes para melhor refletirem sua marca e proposta.
    3. Não foque nos pixels, o objetivo aqui é validar que os requisitos do PRD estão traduzidos em funções na tela.

    Entregável: Link do Figma com as telas base geradas pela IA inserido em uma nova página do seu MkDocs.

    Tarefa 3 — Validação de "Corredor"¶

    Duração estimada: 20 min Formato: Troca entre equipes

    Mostre seu protótipo para um colega de outro grupo (seu "usuário teste") e observe: 1. Ele entendeu o que o produto faz em menos de 10 segundos? 2. Ele identificou onde estão as funcionalidades principais? 3. A solução gerada pela IA realmente ataca o problema do seu PRD?

    Entregável: Um parágrafo com o principal aprendizado/crítica recebida.

    Encerramento¶

    Hoje vimos como a IA pode acelerar a validação de ideias. Na próxima aula (23/03/2026), vamos aprender a medir o interesse real através da criação de Landing Pages de Teste e Métrica de Fumaça (Smoke Test).

    Tarefa de Casa — Evoluir o MkDocs do Produto

    Refine seu protótipo completo utilizando o Figma Maker (Online), englobando os requisitos validados hoje.

    A entrega final: Publique o link do protótipo no MkDocs do grupo. Aproveite para criar uma página chamada 02-prototipo.md onde vocês descrevem as 3 funcionalidades core e mostram o link do Figma.

    Traga o MkDocs atualizado e hospedado no Pages para a próxima aula.
